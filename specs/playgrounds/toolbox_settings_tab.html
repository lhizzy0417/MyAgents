md-merge

~/.myagents/tools/md-merge · v1.0.0 · 注册于 2026-06-12

合并多个 Markdown 文件为一个文档。用户要把多篇 markdown 笔记/章节/报告拼成一个文件、或要求"把这些 md 合成一份"时使用；也适用于把目录下所有 .md 汇总成单文档再做后续转换。单文件场景不要用（直接处理即可）。快速参考：md-merge a.md b.md --out merged.md。首次使用前先运行 md-merge readme；机器可读输出加 --json。

何时使用

把多个 Markdown 文件按顺序拼接为一个文档：多章节笔记汇总、报告合并、为"转 docx/pdf"做前置准备。不适用：单文件转换、需要按标题重排内容的场景。

快速开始

md-merge ch1.md ch2.md --out book.md

参数

<files...> 输入文件，按给定顺序拼接 · --out 输出路径（必填）· --separator hr（默认）| blank · --json 结构化输出

失败模式

退出码 2：缺 --out / 输入文件不存在；退出码 1：写出失败。错误均以 JSON 形式给出原因与修复建议。

产物回流

产物绝对路径打印在 stdout；AI 会在回复中引用该文件，IM 会话中通过 send-media 发送。

video-brief

~/.myagents/tools/video-brief · v1.0.0 · 注册于 2026-06-12

用多模态大模型 API 理解视频内容并产出摘要。用户给一个视频文件要"总结/理解/提取要点"时使用；仅支持本地视频文件路径输入。需要先配置 VIDEO_API_KEY。处理时长与视频长度相关，分钟级视频约需 1-3 分钟。首次使用前先运行 video-brief readme；机器可读输出加 --json。
VIDEO_API_KEY

未配置时工具会以退出码 3 提示 AI 引导你补齐，不会静默失败

何时使用

对本地视频做内容理解：总结、要点提取、按问题聚焦分析。不适用：纯图片理解、需要逐帧时间戳的精细分析。

快速开始

video-brief demo.mp4 --question "总结要点"

失败模式

退出码 3 = 未配置 VIDEO_API_KEY；退出码 2 = 文件不存在/格式不支持/超 200MB；退出码 1 = 上游 API 错误（401 需重设 Key，429 限流稍后重试）。

产物回流

结果为文本摘要（stdout），AI 整理进回复正文；要求落盘时写入工作区并在回复中引用。
